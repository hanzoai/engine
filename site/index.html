Overview

What is Hanzo Engine

Cloud-native LLM inference engine built in Rust. Serves foundational models at scale with CUDA, Metal, and CPU backends. OpenAI-compatible API. Paged attention, continuous batching, speculative decoding, and quantization out of the box. Production-ready.

Capabilities

Features

60+ Architectures

Llama, Qwen3, Phi, Gemma, Mistral, DBRX, Starcoder2, and more. Load any supported model from HuggingFace or local files.

GPU Accelerated

CUDA 12+, Metal, CPU with AVX2/AVX-512. Automatic backend selection based on detected hardware.

Production Ready

Paged attention, continuous batching, speculative decoding, quantization (GGUF, GPTQ, AWQ, EXL2). Built for throughput.

OpenAI Compatible

Drop-in replacement API. /v1/chat/completions, /v1/completions, /v1/embeddings. Works with existing clients.

Quantization

GGUF, GPTQ, AWQ, EXL2, HQQ, and ISQ (In-Situ Quantization). Quantize on load without pre-processing.

Multi-GPU

Tensor parallelism across multiple GPUs. Automatic device mapping for models that exceed single GPU memory.

Get Started

Quick Start

# Install
curl -sSL https://engine.hanzo.ai/install.sh | sh

# Or via cargo
cargo install hanzo-engine

# Serve a model
hanzo-engine serve --model zenlm/zen4 --port 8000

# With CUDA
hanzo-engine serve --model zenlm/zen4 --port 8000 --features cuda

Internals

Architecture

Paged Attention

PagedAttention v2 for efficient KV cache management. Serves more concurrent requests with less GPU memory.

Continuous Batching

Dynamic batch scheduling that adds and removes sequences on the fly. No wasted compute on padding.

Speculative Decoding

Draft model acceleration. Use a smaller model to propose tokens, verified by the target model in parallel.

Tensor Parallelism

Split model layers across multiple GPUs. Serve models larger than single GPU memory with minimal overhead.

ISQ (In-Situ Quantization)

Quantize models on load. No pre-processing step required. Load BF16 weights and serve at INT4/INT8.

Flash Attention

Flash Attention v2 and v3 (Hopper) support. Fused kernels for maximum throughput on supported GPUs.

Infrastructure

Deployment

Docker
docker run -p 8000:8000 --gpus all ghcr.io/hanzoai/engine:latest \
  serve --model zenlm/zen4 --port 8000
Kubernetes
apiVersion: apps/v1
kind: Deployment
metadata:
  name: hanzo-engine
spec:
  replicas: 2
  selector:
    matchLabels:
      app: hanzo-engine
  template:
    metadata:
      labels:
        app: hanzo-engine
    spec:
      containers:
      - name: engine
        image: ghcr.io/hanzoai/engine:latest
        args: ["serve", "--model", "zenlm/zen4", "--port", "8000"]
        ports:
        - containerPort: 8000
        resources:
          limits:
            nvidia.com/gpu: 1
Edge vs Engine

Choose Your Runtime

Hanzo Engine is for cloud servers and production API serving. Hanzo Edge is for on-device and in-browser inference.
